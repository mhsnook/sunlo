# v0.18 Testing Plan — 2026-03-31

## Overview

This release batch from `next` → `main` includes 7 migrations and several new features:

1. **Notifications center** — bell icon, notification list, mark-as-read, realtime subscription
2. **Review answer mode** — 2-button vs 4-button, cascading profile → deck override
3. **Preferred translation language** — per-deck override
4. **Display preferences** — font preference + global review answer mode
5. **Auto-like on create** — auto-upvote content you create, daily recount cron
6. **phrase_stats extraction** — decoupled from phrase_meta for cleaner views
7. **Materialized meta_language** — better read performance for language stats
8. **count_learners RLS fix** — phrase_stats now queries base tables, not RLS-filtered views

## RLS / Security Audit

**Result: PASS (Grade A)**

All 22 tables with uid columns have RLS properly enabled. Key findings:

| Area                                     | Status | Notes                                                                            |
| ---------------------------------------- | ------ | -------------------------------------------------------------------------------- |
| `notification` table RLS                 | PASS   | SELECT/UPDATE for own uid only. No INSERT/DELETE grants to users (triggers only) |
| Notification triggers (SECURITY DEFINER) | PASS   | All 4 validate `uid <> actor_uid`, retrieve correct recipient                    |
| `phrase_stats` view (RLS fix)            | PASS   | Correctly bypasses RLS for aggregate counts (intentional)                        |
| `user_deck_plus` / `user_card_plus`      | PASS   | `security_invoker='true'` enforces user-only data                                |
| `auto_upvote` triggers                   | PASS   | Uses `NEW.requester_uid` / `NEW.uid` from INSERT context                         |
| `recount_all_upvotes()` SECURITY DEFINER | PASS   | Required for pg_cron, operates on public upvote tables                           |
| `review_answer_mode` columns             | PASS   | Protected by existing RLS on user_profile and user_deck                          |

**No security issues found.** One optional enhancement: consider adding a DELETE policy on notification for user soft-delete.

## What's NOT Tested Yet (Future Work)

- **Realtime notification delivery** — `useNotificationsRealtime` subscribes to Postgres changes; needs multi-actor scenetest with `switch-devices` (not yet merged in scenetest)
- **Preferred translation language** — the review card filtering by preferred lang is hard to assert in scenetest without useTestEffect; good candidate for inline assertions
- **Auto-like backfill** — the migration backfills existing content upvotes; would need a DB integration test
- **`recount_all_upvotes()` cron** — runs at 4am UTC; could add a vitest that calls the function directly via supabase client
- **Materialized view refresh** — `refresh_meta_language()` and `refresh_phrase_search_index()` are trigger-based; verify after migration with a DB query
